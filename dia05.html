Como o Computador Vê?

Para nós, uma imagem é uma foto. Para o computador, é uma matriz numérica.

[ 255, 000, 000 ]   ← vermelho puro
[ 000, 255, 000 ]   ← verde puro
[ 000, 000, 255 ]   ← azul puro

Cada pixel armazena a intensidade de Red, Green e Blue (0–255). A IA aprende a partir desses números.

Filtros no Processamento de Imagens

Antes das redes neurais, e dentro delas, filtros são aplicados sobre a imagem para realçar características:

Detecção de Bordas

Destaca contornos e limites de objetos — útil para segmentação.

Suavização (Blur)

Remove ruídos da imagem, facilitando análise posterior.

Nitidez (Sharpen)

Realça detalhes finos — útil em OCR de documentos.

Escala de Cinza

Reduz dimensionalidade; muitas tarefas não precisam de cor.

Filtros = transformações matemáticas aplicadas pixel a pixel (ou em janelas). CNNs aprendem os melhores filtros automaticamente.

Redes Neurais Convolucionais (CNN)

A arquitetura que dominou a visão computacional por mais de uma década.

  • Convolução: Desliza filtros sobre a imagem e gera mapas de características.
  • Pooling: Resume os mapas, reduzindo tamanho e preservando o essencial.
  • Camadas Profundas: Filtros simples → bordas → formas → partes → objetos.

Vision Transformers (ViT) e Modelos Multimodais

Arquitetura mais recente que aplica a ideia dos Transformers (base do GPT) para imagens.

  • ViT: Divide a imagem em patches (pedaços) e os trata como tokens de texto. Aprende relações globais.
  • Vantagem sobre CNN: Captura contexto de longo alcance — reconhece que o "olho" e a "pata" pertencem ao mesmo animal.
  • Modelos Multimodais: Combinam imagem + texto. Exemplos: GPT-4o, Florence da Microsoft.
Exemplo multimodal: Você envia uma foto de um produto danificado e pergunta "Qual o defeito?". O modelo processa a imagem e a pergunta juntos.

IA Generativa para Criação de Imagens

Além de analisar imagens, a IA moderna consegue criá-las a partir de texto.

  • Diffusion Models: Partem de ruído aleatório e, passo a passo, "removem" ruído guiados por um prompt. Base do DALL-E e Stable Diffusion.
  • DALL-E 3 (Azure OpenAI): Geração de imagens via API integrada ao Azure.
  • Inpainting: Editar apenas parte de uma imagem com IA.
Atenção para a prova: Geração de imagem = IA Generativa. Análise de imagem = Azure AI Vision. São serviços e conceitos distintos.

Azure AI Vision: Image Analysis

Serviço pronto (sem treino) para analisar imagens via API.

  • Captioning: Descrição automática ("Um homem andando de skate numa rua").
  • Dense Captioning: Descreve múltiplas regiões da mesma imagem.
  • Tagging: Extrai palavras-chave ("Skate", "Esporte", "Asfalto").
  • Object Detection: Localiza objetos com bounding boxes.
  • Smart Cropping: Gera thumbnails mantendo o ponto focal.
Tudo via chamada REST — não é necessário treinar nada.

Azure AI Face: Detecção Facial

Serviço especializado em rostos humanos em imagens e vídeos.

  • Face Detection: Localiza rostos na imagem (posição, tamanho).
  • Face Verification: Essa pessoa no selfie é a mesma do documento? (comparação 1:1).
  • Face Identification: De quem é esse rosto? (comparação 1:N com uma lista).
IA Responsável: A Microsoft restringiu atributos como inferência de emoções e idade por risco de discriminação. Reconhecimento facial de alta precisão exige aprovação de uso responsável.

Extração de Informações com IA

Empresas acumulam enorme volume de dados não estruturados: notas fiscais, contratos, formulários digitalizados.

  • Dados não estruturados não estão em bancos de dados — estão em PDFs, imagens, e-mails.
  • Extração de Informações: Processo de identificar e capturar valores específicos desses documentos.
  • Envolve dois passos principais: OCR (ler o texto) e Mapeamento de Campos (entender a estrutura).
Exemplo: Uma seguradora recebe 5.000 sinistros por dia em PDF. A IA lê cada documento e popula automaticamente os campos no sistema.

OCR: Reconhecimento Óptico de Caracteres

Converte imagens de texto em strings pesquisáveis e editáveis.

  • Read API (Azure Vision): Otimizada para documentos densos — PDFs, livros, fotos de placa.
  • Suporta letra de forma e letra manuscrita (handwriting).
  • Retorna posição de cada palavra na imagem (bounding boxes).
  • Suporta dezenas de idiomas.

Extração e Mapeamento de Campos

O OCR lê o texto. O mapeamento entende o que aquele texto representa.

NívelO que fazExemplo
OCRLê pixels e gera texto"R$ 350,00"
LayoutIdentifica tabelas e seçõesLinha 3, coluna "Valor"
MapeamentoAssocia texto ao campo corretocampo total_nota = R$ 350,00
O mapeamento transforma dados brutos em registros estruturados prontos para um banco de dados ou API.

Azure Vision: Extração de Texto

A Read API do Azure Vision é o serviço base de OCR na plataforma.

  • Processa imagens (JPEG, PNG, TIFF) e PDFs de até 2 GB.
  • Retorna texto com layout: linhas, palavras e polígonos de posição.
  • Ideal quando o objetivo é apenas extrair o texto sem interpretar campos.
Caso de uso: Digitalizar acervo histórico de jornais impressos para criar um buscador de notícias.

Azure Content Understanding

Serviço mais recente: extração multimodal de conteúdo complexo.

  • Processa documentos que misturam texto, imagens e tabelas de forma integrada.
  • Entende contexto semântico — não só o texto, mas o que ele significa dentro do layout.
  • Suporta também extração de áudio e vídeo (transcrição + análise).
Diferencial: Onde o OCR tradicional lê texto linha a linha, o Content Understanding entende que aquele texto é um título de seção, não um valor de campo.

Azure AI Document Intelligence

Antigo "Form Recognizer". Especializado em formulários e documentos estruturados.

  • Combina OCR + compreensão de layout + mapeamento de campos.
  • Retorna JSON com pares chave-valor já associados ao campo correto.

OCR diz:

"TOTAL   R$ 350,00"

Document Intelligence diz:

campo InvoiceTotal = R$ 350,00

Modelos Pré-treinados

Prontos para usar, sem necessidade de treinamento:

  • Invoice: Notas fiscais — extrai Fornecedor, Data, Itens, Total.
  • Receipt: Recibos de compras no varejo.
  • ID Document: Passaportes e carteiras de motorista (campos padronizados).
  • Business Card: Cartões de visita — nome, telefone, e-mail.
  • Layout: Estrutura genérica — tabelas e parágrafos sem campos fixos.
Para a prova: "Passaporte" → ID Document. "Nota fiscal" → Invoice.

Modelos Customizados no Document Intelligence

Quando os modelos prontos não atendem ao seu formulário exclusivo:

  1. Faça upload de 5 ou mais exemplos preenchidos do seu formulário.
  2. Use a ferramenta visual para rotular os campos (Ex: "campo_cnpj", "campo_data").
  3. Treine o modelo — o Azure aprende o layout específico do seu documento.
  4. Publique e consuma via API como qualquer modelo pré-treinado.
Exemplo real: Formulário de sinistro de seguradora com layout proprietário. Um modelo customizado aprende exatamente onde fica cada campo.

Azure AI Search: Mineração de Conhecimento

O problema do "dark data": empresas têm milhões de PDFs, imagens e documentos Word que ninguém consegue pesquisar.

  • O Azure AI Search indexa todo esse conteúdo e torna possível buscas inteligentes.
  • Combina indexação tradicional com habilidades de IA (OCR, NER, tradução).
  • Base de soluções RAG (Retrieval Augmented Generation) com Azure OpenAI.
Caso de uso: Escritório de advocacia com 10 anos de contratos em PDF. Com AI Search, o advogado pesquisa "cláusulas de rescisão" e encontra todos os documentos relevantes.

Knowledge Store e Search Index

Dois destinos para os dados enriquecidos pelo pipeline:

Search Index

Banco JSON otimizado para busca. Usado por chatbots, apps de busca e RAG com GPT.

Knowledge Store

Salva projeções em tabelas SQL ou Blob JSON para análise no Power BI ou relatórios.

Busca Semântica vs Palavra-chave: Busca por palavra-chave encontra "carro". Busca semântica entende que "veículo de quatro rodas" também é relevante.

Quiz Rápido — Visão Computacional

Q1: Você precisa contar quantas pessoas entram numa loja, sem identificar quem são. Qual serviço usar?

  • A) Face API (Identification)
  • B) Azure AI Vision (Object Detection)
  • C) Custom Vision (Classification)
Resposta: B. Object Detection detecta e conta instâncias de "Pessoa" sem identificar a identidade.

Q2: Qual arquitetura divide a imagem em patches e usa mecanismo de atenção?

  • A) CNN     B) ViT     C) OCR Engine
Resposta: B. Vision Transformer (ViT) trata patches de imagem como tokens.

Quiz Rápido — Extração de Informações

Q1: Qual serviço é mais adequado para extrair dados de passaportes internacionais?

  • A) Read API    B) Azure Content Understanding    C) Document Intelligence (ID Document)
Resposta: C. O modelo ID Document é pré-treinado para passaportes e carteiras de motorista.

Q2: No pipeline do AI Search, onde ocorre a transformação de imagem escaneada em texto pesquisável?

  • A) Data Source    B) Skillset    C) Index
Resposta: B. O Skillset contém a habilidade de OCR que extrai texto de imagens.
