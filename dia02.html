A Anatomia de um Dataset

Todo modelo de ML começa com uma tabela. Vamos dar nomes técnicos às colunas.

Metragem (m²) Quartos Bairro Preço (R$)
80 2 Centro 350.000
120 3 Jardins 800.000
Y = f(X)
  • X (Features/Recursos): As colunas azuis (Metragem, Quartos, Bairro). São as variáveis independentes.
  • Y (Label/Rótulo): A coluna vermelha (Preço). É a variável dependente, o que queremos prever.

As 3 Grandes Categorias de Aprendizado

Dependendo se temos o Y (Label) ou não, o tipo de aprendizado muda.

  1. Aprendizado Supervisionado: Nós temos o Gabarito (Label).
    • Ex: Regressão e Classificação.
  2. Aprendizado Não Supervisionado: Não temos o Gabarito. A máquina deve achar estrutura sozinha.
    • Ex: Clustering (Agrupamento).
  3. Aprendizado por Reforço: A máquina aprende por tentativa e erro (recompensa/punição).
    • Ex: Robô aprendendo a andar ou jogar Xadrez.

O Ciclo de Vida: Treino e Teste

Jamais treinamos o modelo com 100% dos dados. Por que?

Analogia: Se o professor der as mesmas perguntas da aula na prova, o aluno apenas memoriza, não aprende. Isso se chama Overfitting.
Dados de Treino (70%)
Para a máquina estudar
Dados de Teste (30%)
Para a prova final

O Azure ML faz essa divisão (Split) automaticamente na maioria dos casos.

1. Regressão

O objetivo é prever um valor numérico contínuo.

O algoritmo tenta traçar a linha que passa mais perto de todos os pontos.

  • Algoritmos comuns: Regressão Linear, Árvore de Decisão, Random Forest.

Avaliando a Regressão: O Erro

Como sabemos se a linha está boa? Medimos a distância dos pontos até a linha.

  • MAE (Mean Absolute Error): A média simples dos erros.
    "Errei o preço das casas em média por R$ 5.000."
  • RMSE (Root Mean Squared Error): Média quadrática.
    Penaliza erros grandes. Se errar feio em uma casa, o RMSE sobe muito.
  • R² (R-Squared): Coeficiente de Determinação (0 a 1).
    Quanto mais perto de 1, melhor o modelo explica os dados.
Para a prova: Menor MAE/RMSE é bom. Maior R² é bom.

2. Classificação Binária

Prevê uma entre duas categorias mutuamente exclusivas.

  • É Spam ou Não é?
  • O paciente tem Diabetes ou Não?
  • A transação é Fraude ou Legítima?

O algoritmo calcula uma Probabilidade (entre 0 e 1). Se for > 0.5, é Classe A. Se < 0.5, é Classe B.

Algoritmo Famoso: Regressão Logística (apesar do nome "regressão", é usado para classificação).

A Matriz de Confusão

Acurácia não é tudo! Precisamos ver onde o modelo errou.

Previ: SIM Previ: NÃO
Real: SIM Verdadeiro Positivo (TP)
Acertei!		 Falso Negativo (FN)
Errei (Perigo!)
Real: NÃO Falso Positivo (FP)
Alarme Falso		 Verdadeiro Negativo (TN)
Acertei!

Precisão vs. Recall (Revocação)

Dependendo do negócio, escolhemos o que priorizar.

  • Precisão: De todos que eu chamei de "Doentes", quantos realmente eram?
    Meta: Não assustar pacientes saudáveis (Evitar Falso Positivo).
  • Recall: De todos os "Doentes" que existem, quantos eu encontrei?
    Meta: Não deixar ninguém morrer sem tratamento (Evitar Falso Negativo).
Em saúde e segurança, geralmente priorizamos o Recall. Em filtros de spam, priorizamos a Precisão.

3. Classificação Multiclasse

Quando temos mais de duas opções possíveis.

  • Classificar gêneros de filmes: Ação, Comédia, Terror, Drama.
  • Classificar sentimentos: Feliz, Triste, Raiva, Neutro.

O modelo gera uma probabilidade para cada classe e escolhe a maior.

Ação: 0.10
Comédia: 0.85 (Vencedor)
Terror: 0.05

4. Clustering (Agrupamento)

Aprendizado Não Supervisionado. O algoritmo organiza a bagunça.

Algoritmo K-Means:

  1. Você define K (número de grupos, ex: 3).
  2. O algoritmo coloca 3 pontos aleatórios (centróides).
  3. Cada dado se junta ao centróide mais próximo.
  4. O centróide se move para o centro do grupo.
  5. Repete até estabilizar.
Uso: Segmentação de mercado. "Descubra quais são os tipos de clientes que tenho".

Deep Learning: O que muda?

Enquanto o ML clássico (Regressão/K-Means) é bom para planilhas, ele falha com dados complexos como Imagens e Áudio.

  • Redes Neurais Artificiais (ANN): Inspiradas nos neurônios biológicos.
  • Camadas (Layers): Camada de entrada -> Camadas Ocultas (Processamento) -> Camada de Saída.
  • Deep: Significa que temos muitas camadas ocultas.

Aprofundando no Automated ML

O AutoML democratiza a IA. Ele automatiza:

  1. Featurization: Limpa dados, trata valores nulos, normaliza escalas.
  2. Seleção de Modelo: Testa XGBoost, Linear, Random Forest, etc.
  3. Hiperparâmetros: Ajusta a "sintonia fina" de cada modelo.
Na Prova: Se o cenário pede "criar um modelo rápido com pouco conhecimento de código", a resposta é AutoML.

Aprofundando no Designer

Ferramenta visual baseada em Pipelines.

  • Cada caixinha é um Módulo.
  • Você conecta a saída de um módulo (ex: "Clean Missing Data") na entrada de outro (ex: "Train Model").
  • Split Data: Módulo obrigatório para dividir treino/teste.
  • Evaluate Model: Módulo final para gerar métricas (Acurácia, Recall).

Compute Targets (Onde roda?)

Você não treina modelos no seu laptop. Você usa a nuvem.

  • Compute Instance: Uma VM única na nuvem para você desenvolver (seu PC na nuvem).
  • Compute Cluster: Várias VMs trabalhando juntas. Escala para zero quando não usado (Economia!). Usado para treinos pesados e AutoML.
  • Inference Clusters (AKS): Kubernetes para colocar o modelo em produção (Real-time).

Gerenciamento de Dados

O Azure ML separa o armazenamento da lógica.

  • Datastore: A conexão segura com o Azure Storage (Blob, SQL). Guarda as senhas.
  • Dataset: Uma referência aos dados versionada.
    Ex: "Vendas_2025_v1". Se os dados mudarem, criamos a v2.

Colocando em Produção (Deploy)

Treinar é só metade do trabalho. O modelo precisa ser usado.

  • Real-time Endpoint: O app envia os dados e recebe a resposta na hora (Ex: Fraude no cartão).
  • Batch Endpoint: Processa milhões de linhas de uma vez durante a madrugada (Ex: Previsão de estoque semanal).

Cenário Prático 1: Aluguel de Bicicletas

Problema: Quantas bicicletas preciso colocar na estação amanhã?

  • Features: Previsão do tempo, Feriado (Sim/Não), Dia da semana.
  • Label: Quantidade alugada (Número).
  • Tipo: Regressão.
  • Métrica: RMSE (Quero errar o mínimo de bicicletas possível).

Cenário Prático 2: Diagnóstico de Diabetes

Problema: O paciente tem risco de diabetes?

  • Features: Idade, Peso, Pressão Sanguínea, Histórico Familiar.
  • Label: 1 (Sim) ou 0 (Não).
  • Tipo: Classificação Binária.
  • Métrica: Recall (Não quero deixar passar nenhum doente).

Quiz Rápido

Você trabalha numa fábrica de chocolates. Quer agrupar os bombons por tamanho e peso para criar caixas sortidas, mas não tem rótulos prévios.

Qual algoritmo usar?

  • A) Regressão Linear
  • B) K-Means Clustering
  • C) Regressão Logística
  • D) AutoML
Resposta: B. É um problema de agrupamento sem supervisão.

Quiz Rápido

Seu modelo de previsão de preços de casas tem um R² de 0.12.

Isso é bom ou ruim?

  • A) Excelente, quanto menor melhor.
  • B) Ruim, R² deve ser próximo de 1.
  • C) Indiferente, R² é para classificação.
Resposta: B. R² perto de 0 significa que o modelo não aprendeu nada.

Pegadinhas do Exame AI-900

  • Confundir Classificação com Clustering: Lembre-se, Classificação TEM rótulos (classes conhecidas). Clustering NÃO TEM.
  • Confundir Regressão com Regressão Logística: Regressão Logística é para Classificação! (Sim, o nome confunde).
  • Esquecer do Split: Nunca avalie o modelo com os dados de treino.
